The Opportunity

Why Vedic Maths + Coding?

Both Vedic Mathematics and Computer Programming share the same DNA — they are systems of precise, pattern-based rules applied to solve problems efficiently. Teaching them together creates a multiplier effect on a child's logical reasoning.

🧩

Same Cognitive Skills

Vedic sutras train the brain to look for patterns, shortcuts, and elegant solutions — exactly what a programmer does. Children who learn Vedic Maths first pick up coding significantly faster because they're already thinking algorithmically.

🇮🇳

Rooted in Indian Heritage

Vedic Mathematics comes from the Atharva Veda. Taught alongside modern coding, it gives Mumbai students a pride of identity — showing that India's ancient scholars were the world's first programmers, writing algorithms in Sanskrit.

⚡

Faster Mental Computation

Kids who practice Vedic techniques compute 3–5× faster mentally. In coding, this translates to faster debugging, quicker mental simulation of code logic, and the ability to estimate results before running a program.

🔁

Sutras ARE Algorithms

The word sutra means "thread" — a rule you follow step by step. That's exactly what an algorithm is. Each Vedic sutra is a real algorithm; translating them into code makes both maths and coding instantly more concrete and understandable.

🏫

Perfect for Mumbai Schools

Mumbai's CBSE and ICSE schools already cover basic Vedic methods in Grade 6–8 syllabi. This program builds on what teachers already know, requires no new equipment, and aligns perfectly with NEP 2020's emphasis on coding and critical thinking.

👩‍🏫

Teacher-Friendly Design

Every session comes with a complete teacher guide, student worksheets, digital resources, and step-by-step instruction scripts. A computer teacher with no Vedic Maths background can confidently deliver Week 1 after a single half-day training session.

The Bridge

5 Ways Vedic Maths Directly Powers Coding

These are not metaphors — these are direct, concrete mappings that we use in every lesson. Each Vedic sutra becomes a coding concept children can see, build, and run.

01
Ekadhikena Purvena
"By one more than the previous one"

The simplest Vedic sutra — add 1 to what came before — is literally a counter loop. Children who understand this sutra immediately grasp why computers count the way they do.

📐 Vedic Maths
Multiply 1/19 = 0.052631...
Each digit: previous digit ×2
Pattern: 1 → 2 → 4 → 8 → ...

Squaring: 25² = (2×3) | 25 = 625
35² = (3×4) | 25 = 1225
45² = (4×5) | 25 = 2025
💻 In Scratch / JS
This IS a loop!
-- Scratch -- repeat 10 change score by 1 end // JavaScript for (let i = 1; i <= 10; i++) { console.log(i); }
02
Nikhilam Navatashcaramam Dashatah
"All from 9, the last from 10"

This sutra computes complements from the nearest base (10, 100, 1000). In programming, this is identical to the concept of variables that store intermediate values and the subtraction operator.

📐 Vedic Maths
97 × 96 = ?
Base = 100
Deficit of 97 = -3
Deficit of 96 = -4
Left: 97-4 = 93 (or 96-3)
Right: (-3)×(-4) = 12
Answer: 9312
💻 In Scratch / JS
Variables store the "deficits"
let a = 97, b = 96; let base = 100; let defA = base - a; // 3 let defB = base - b; // 4 let left = a - defB; // 93 let right = defA * defB; // 12 let ans = left*100 + right; // 9312
03
Urdhva Tiryakbhyam
"Vertically and Crosswise"

The Vedic method for multi-digit multiplication — cross-multiply and carry. This maps directly to nested loops, where an inner loop runs inside an outer loop, processing every combination of digits.

📐 Vedic Maths
23 × 41 = ?
Step 1 (vertical right): 3×1 = 3
Step 2 (cross): 2×1 + 3×4 = 14, write 4, carry 1
Step 3 (vertical left): 2×4 = 8 + carry 1 = 9
Answer: 943
💻 In JS — Nested Loops
Outer loop = tens, inner = units
// Multiplication table for (let i = 1; i <= 10; i++) { for (let j = 1; j <= 10; j++) { console.log(i + "×" + j + "=" + (i*j)); } }
04
Gunita Samuchaya
"The product of the sum equals the sum of the product"

The Vedic technique of casting out 9s — computing a digital root to verify calculations. In coding, this is identical to testing and debugging: running a quick check to verify your program's output is correct.

📐 Vedic Verification
Check: 47 × 83 = 3901?
Digital root of 47 = 4+7 = 11 → 2
Digital root of 83 = 8+3 = 11 → 2
Product of roots: 2×2 = 4
Digital root of 3901 = 3+9+0+1 = 13 → 4 ✓
Answer is verified correct!
💻 In JS — Modulo (%) Operator
Digital root uses mod 9!
function digitalRoot(n) { return n === 0 ? 0 : 1 + (n - 1) % 9; } // Verify multiplication let a = 47, b = 83; let result = a * b; // 3901 console.log(digitalRoot(a) * digitalRoot(b) % 9 === digitalRoot(result) % 9); // true ✓
05
Anurupyena
"Proportionality"

The Vedic principle of scaling — if you know 1×, you know 2×, 3×, any multiple. In coding, this maps directly to functions: write the rule once, call it with any input, and the proportional result scales automatically.

📐 Vedic Maths
Know 11×7 = 77?
Then instantly: 110×7 = 770
1100×7 = 7700
0.11×7 = 0.77
Proportional scaling — same rule, different input.
Also: CSS percentage layouts!
1 col = 100%, 2 col = 50%, 3 col = 33.33%
💻 In JS — Functions / CSS
Write once, scale infinitely
// One function, any scale function multiply(base, factor) { return base * factor; } multiply(11, 7); // 77 multiply(110, 7); // 770 /* CSS proportionality */ .col-3 { width: 33.33%; } .col-2 { width: 50%; } .col-1 { width: 100%; }

Grades 4–5: Vedic Maths + Scratch

16 weeks, once a week, 45 minutes per session. Each week pairs a Vedic technique with a Scratch concept — then students immediately build something with it.

🐱 Scratch Track
Phase 1: Foundations Weeks 1–4
Week 01
Welcome to T.H.Y.N.K! What are Vedic Maths & Scratch?
📐 Vedic
Introduction to the 16 Sutras. What is a sutra? (A rule!) Examples: multiplying by 11, squaring numbers ending in 5. Show how ancient Indians computed without calculators.
💻 Scratch
Open Scratch. Meet Sprites, the Stage, and Blocks. Run a "Hello World!" — make the cat say "Namaste!" and wave. Understand what a program is.
🎯 Outcome: Students can explain what a sutra and a program have in common — both are step-by-step rules.
Week 02
Ekadhikena Purvena — Counting & Motion Blocks
📐 Vedic
Sutra: "By one more than the previous one." Practice: 1/19, 1/29 using this pattern. Spot doubling sequences: 1, 2, 4, 8, 16... What comes next?
💻 Scratch
Motion blocks: move 10 steps, turn 15 degrees. Make sprite walk by pressing arrow keys. Each key press = "one more step." Worksheet: match the sutra step to the code step.
🎯 Outcome: Students see that "add 1 more" is the same action in Vedic counting and in making a sprite move. Mini-project: Walking cat that counts steps.
Week 03
Number Patterns → The Repeat Loop
📐 Vedic
Explore repeating patterns in multiplication tables. 9's table: 9, 18, 27, 36... digit sum always = 9! Eleven's table: digit mirror pattern 11, 22, 33... Spot the pattern, predict the next.
💻 Scratch
The repeat block — let the computer repeat instead of you. Repeat 10: move 20, turn 36° → draws a perfect circle! Connection: the multiplication pattern IS the loop.
🎯 Outcome: Students build a spiral/polygon using repeat loops. They explain why 360÷10=36° using Vedic proportional thinking.
Week 04
Multiplying by 11 → Variables & Answers
📐 Vedic
Trick: 23×11 = 2_3, middle = 2+3 = 5 → 253. 45×11 = 4_5, 4+5=9 → 495. What if sum > 9? 78×11: 7+8=15, carry 1 → 858. Practice 10 examples mentally.
💻 Scratch
Sensing blocks: ask and answer. Variables: Number1, Result. Build a ×11 calculator: ask a 2-digit number, compute it using the Vedic trick as separate variable steps, say the answer.
🎯 Outcome: Students build their first calculator in Scratch. They see how the Vedic steps (split digits → add middle → join) become variable operations in code.
Phase 2: Building Tools Weeks 5–8
Week 05
All from 9, Last from 10 → Subtraction & Score Bars
📐 Vedic
Nikhilam sutra: 1000−386 = ? Take each digit from 9, last from 10: 6,1,4 → 614. Instant! Practice: 10000−2847, 100−73. This is complement arithmetic — same as computers use!
💻 Scratch
Build a health bar game. Variable: Lives = 3. Each time a ball touches the sprite: change Lives by -1. If Lives = 0: game over. This IS "all from [max], last from [max]" in code.
🎯 Outcome: Students build a working game with a live health counter. They explain the connection between the Nikhilam sutra and the Lives variable.
Week 06
Squaring Numbers Ending in 5 → Formulas & Functions
📐 Vedic
Any number ending in 5: n×(n+1) | 25. 35² = 3×4 | 25 = 1225. 75² = 7×8 | 25 = 5625. 105² = 10×11 | 25 = 11025. No multiplication needed — pure pattern recognition.
💻 Scratch
Advanced calculator: Ask for a number ending in 5. Compute using the Vedic formula (store tens digit in variable, multiply by tens+1, join 25). Display result. Compare with direct multiply to verify!
🎯 Outcome: Students build a Vedic squaring calculator and experience the joy of getting the same result two different ways — intuitive Vedic vs brute-force multiplication.
Week 07
Near-10 Multiplication → Scoring & Operators
📐 Vedic
Base 10 multiplication: 8×7 = ? Both near 10. 8 is -2 from 10, 7 is -3. Cross: 8-3=5 (or 7-2=5). Product of deficits: 2×3=6. Answer: 56. Try 9×6, 7×6, 8×9 mentally.
💻 Scratch
Scoring system: Green flag starts score=0. Each correct answer: score +10. Each wrong answer: score -5. Use operator blocks (+, -, ×) directly. The deficits in Vedic = the penalty/bonus in the game!
🎯 Outcome: Students build a working quiz with live score tracking using Vedic multiplication as both the subject and the scoring logic.
Week 08 — Mini Project 🏆
Build a Vedic Maths Quiz Game in Scratch
📐 Vedic Content
Quiz covers: ×11 trick, squaring ending in 5, near-10 multiplication, Nikhilam subtraction. 5 questions, timed (30 seconds each). Use all Vedic shortcuts learned so far.
💻 Scratch Features
Must include: Variables (Score, Timer, Lives), Ask/Answer blocks, If/Else for correct/wrong, Repeat loop for countdown, Say block for feedback. Full working game!
🎯 Students share and play each other's quiz games. Peer testing = informal code review! Teachers assess variables, loops, and conditionals.
Phase 3: Deeper Concepts Weeks 9–12
Week 09
Digital Roots → Conditionals (If / Else)
📐 Vedic
Digital root = keep adding digits until single digit. 9→9, 10→1, 11→2, 18→9, 19→1. Rules: if digital root = 9 → divisible by 9. If = 3 or 6 → divisible by 3. Instant divisibility check!
💻 Scratch
If/Else blocks: If (digital root of answer = 9) then say "Divisible by 9!" else say "Not divisible by 9." Build a divisibility checker — the Vedic rule IS the condition!
🎯 Outcome: Students build a divisibility tester. The if/else logic comes naturally because they already know the Vedic rule that goes inside the condition.
Week 10
Proportions (Doubling & Halving) → Sprite Sizing & Looks
📐 Vedic
Anurupyena: proportional scaling. If 12×5=60, then 24×5=120, 6×5=30, 1.2×5=6. Double the number, double the answer. Half the number, half the answer. Apply to areas: 2× length = 4× area.
💻 Scratch
Looks blocks: set size to %, change size by. Make sprite grow when score increases (size = score × 2), shrink when score drops. Show proportional relationship visually. Also: set costume effects proportionally.
🎯 Outcome: Students create an animated "score creature" that visually scales proportionally with the game score — direct visual proof of Anurupyena.
Week 11
Vertically & Crosswise → Nested Loops & Grid Art
📐 Vedic
Urdhva Tiryakbhyam: 2-digit × 2-digit multiplication in 3 steps — right column, cross, left column. Process every position systematically. This is exactly what a nested loop does!
💻 Scratch
Nested repeat loops: outer loop rows (move down 20), inner loop columns (stamp, move right 20). Creates a grid — exactly like filling in a multiplication table position by position. Draw Vedic yantra art!
🎯 Outcome: Students produce a stamp-art grid resembling a Vedic magic square. They narrate: "The outer loop is the tens digit, the inner loop is the units digit."
Week 12
Gunita Samuchaya — Testing & Debugging Like a Vedic Scholar
📐 Vedic
Verification sutra: use digital roots to check any multiplication. 47×83=3901? Root(47)=2, Root(83)=2, 2×2=4. Root(3901)=4. ✓ If they don't match → ERROR. This is a built-in error-detection system!
💻 Scratch
Debugging session: teacher provides 3 broken Scratch projects. Students find bugs using: check variables panel, add say blocks to test values, trace execution step by step. Debugging = Vedic verification mindset.
🎯 Outcome: Students fix all 3 buggy projects. Key insight: "Checking is part of the work, not extra work" — same in Vedic maths and in programming.
Phase 4: Final Project Weeks 13–16
Week 13
Project Planning — Design Your Vedic Game
📐 Vedic
Students choose which Vedic sutra their game will teach. Complete a "Game Design Sheet": what sutra, what question format, how score works, what happens when correct/wrong, what's the win condition.
💻 Scratch
Sketch the Scratch layout: which sprites, which variables, which events. Draw a flowchart: Green Flag → Ask → Check → Score → Win/Lose. Map every box to the Scratch block that will do it.
🎯 Deliverable: Completed Game Design Sheet with Vedic sutra chosen, flowchart drawn, and sprites listed. Teacher approves before building starts.
Week 14
Project Build — Week 1
📐 Vedic
Each student must include at least 2 Vedic sutra questions. The Vedic calculations should be broken into steps matching the code (e.g., three separate variables for the three steps of ×11 trick).
💻 Scratch
Build core game loop: ask question, get answer, check if correct, update score, go to next question. All variables named clearly. Teacher circulates and helps with debugging. Peer pair support encouraged.
🎯 By end of class: Core loop working (even with just 1 question). Students save and share project link.
Week 15
Project Polish — Test, Debug, Beautify
📐 Vedic
Students write a 5-question Vedic Maths test on paper using their chosen sutra. They swap papers with a classmate and solve using the sutra. This reveals if the sutra is correctly represented in their game.
💻 Scratch
Add: start screen, end screen with "Play Again" button, sound effects, custom backdrop, at least 3 questions, timer, and high score. Use a classmate to test — if they're confused, fix the UX. Testing = verification.
🎯 Final game complete, saved, and shared. Each student records a 60-second voice note explaining their sutra and how it appears in their code.
Week 16 — Demo Day 🎉
T.H.Y.N.K Demo Day — Present & Celebrate!
📐 Presentation
Each student presents for 2 minutes: "My game teaches [sutra name]. This sutra means... In my game, you need to... The sutra helps because..." Judges (teachers, parents) ask: "How did Vedic maths help you code this?"
💻 Demo
Live demo of game on screen. Audience plays 1 round. Student explains one piece of code (open the blocks, point to it). Certificate of completion. Games published to class Scratch studio for all to play.
🏆 Students receive T.H.Y.N.K Certificate. Best projects featured in school newsletter. All games archived in school Scratch studio for future students.

Grades 6–7: Vedic Maths + HTML / CSS / JS

16 weeks, once a week, 45 minutes per session. Students build real websites while discovering that HTML, CSS and JavaScript are just Vedic sutras written in a different language.

🌐 Web Track
Phase 1: Structure & Style Weeks 1–4
Week 01
What is a Webpage? HTML = A Sutra for Browsers
📐 Vedic
What is a sutra? A rule with a name, a structure, and a predictable outcome. The Vedic texts contain 16 sutras and 13 sub-sutras — exactly like HTML has tags and attributes. Structure = sutra.
💻 HTML
Write first HTML file in Notepad/VS Code. Tags: <html>, <head>, <body>, <h1>, <p>. Open in Chrome. Students add their name, school, and a Vedic sutra quote. Save, refresh, see changes instantly.
🎯 Outcome: Each student has a personal "about me" webpage live in Chrome — their first HTML file.
Week 02
HTML Lists & Tables — Organising Sutras
📐 Vedic
The 16 Vedic Sutras listed: Ekadhikena, Nikhilam, Urdhva Tiryak... each has a Sanskrit name, English meaning, and use case. Classification and hierarchy — just like HTML elements!
💻 HTML
<ul>, <ol>, <li> for lists. <table>, <tr>, <td> for the multiplication table. Build an HTML page: "The 16 Vedic Sutras" — one table row per sutra with Sanskrit name, meaning, and example.
🎯 Outcome: Students create a well-structured HTML reference page of all 16 Vedic Sutras.
Week 03
CSS Basics — Anurupyena (Proportions) in Style
📐 Vedic
Anurupyena: proportional relationships. If font is 16px, a heading should be 2× = 32px. Line height should be 1.5× font size. Spacing: margin = 1× font, padding = 0.5× font. All proportions flow from one base unit — exactly like Vedic scaling.
💻 CSS
Link CSS file to HTML. Selectors: body, h1, p, table. Properties: color, background-color, font-size, font-family. Students style their Vedic Sutras page. Choose a colour scheme inspired by India: saffron, white, green.
🎯 Outcome: Sutras page is now beautifully styled. Students discover that good CSS proportions feel the same as Vedic proportional thinking.
Week 04
CSS Box Model — Vedic Geometry & Squared Areas
📐 Vedic
Vedic geometry (Sulbasutras — pre-dates Pythagoras!): area of square = side². The 3-4-5 right triangle was known. Box inside box inside box — exactly the CSS box model. Content → padding → border → margin.
💻 CSS
padding, margin, border, width, height. box-sizing: border-box. Build a "Vedic yantra" card layout using nested divs: outer border (margin), decorated frame (border), breathing room (padding), content in centre.
🎯 Outcome: Students create a styled Vedic Yantra card using only CSS box model properties. They narrate each layer using Vedic geometry language.
Phase 2: Making it Work with JavaScript Weeks 5–8
Week 05
Intro to JavaScript — Variables as Unknown Quantities
📐 Vedic
In Vedic algebra (Bija Ganita), unknowns were called "yavat-tavat" (as much as). A variable is an unknown that takes a value. This is identical to algebra AND identical to JavaScript variables. Ancient India had variables!
💻 JS
script tags. let, const. prompt() to get input. alert() to show output. console.log() for testing. First JS program: ask two numbers, multiply them (using the ×1 trick for type conversion), show the result.
🎯 Outcome: Students write their first JavaScript and connect it directly to Vedic algebraic unknowns — variables were not invented in 1950s, they were used in 1000 BCE India.
Week 06
JS Operators — All Four Vedic Operations
📐 Vedic
All four operations and their Vedic shortcuts: + (Sankalana — addition), - (Vyavakalana — subtraction), × (Gunana — near-base method), ÷ (Bhagana — Paravartya division). Show the shortcut method for each, then verify with JS.
💻 JS
+, -, *, /, % (modulo). Math.pow(), Math.sqrt(). Build a 4-operation Vedic calculator HTML page: input two numbers, four buttons (one per operation), result displayed in a div. CSS styled in India tricolour.
🎯 Outcome: Students have a working web calculator. They also know the Vedic Sanskrit name for each operation — connects cultural pride to technical skill.
Week 07
JS Conditionals — Shunyam & Special Cases
📐 Vedic
Sutra: "Shunyam Saamyasamuccaye" — if the sum is the same on both sides, that quantity is zero. Special case detection! Vedic maths has rules that say "IF [condition] THEN [special shortcut] ELSE [general method]."
💻 JS
if / else if / else. Comparison operators: ===, !==, >, <. Build divisibility tester: input a number, JS checks digital root and says "Divisible by 3/9/neither." The Shunyam condition becomes an if-statement.
🎯 Outcome: Students see if/else as the direct equivalent of Vedic "special case rules" — the code structure was already implicit in the sutra.
Week 08 — Mini Project 🏆
Build a Full Vedic Maths Calculator Webpage
📐 Vedic
Integrates: ×11 shortcut, squaring-ending-5, Nikhilam subtraction, near-10 multiplication, digital root check. Calculator has a "Show Vedic Steps" button that reveals the sutra's intermediate steps alongside the answer.
💻 HTML + CSS + JS
HTML: input fields, buttons, result area, steps area. CSS: fully styled, responsive, uses flexbox for layout. JS: event listeners, if/else for choosing which sutra to apply based on the numbers entered. Three files, one cohesive product.
🎯 Deliverable: A hosted, working Vedic Maths calculator with step-by-step Vedic explanation panel. Teachers save these as portfolio pieces.
Phase 3: Loops, Functions & Advanced Ideas Weeks 9–12
Week 09
JS Loops — Urdhva Tiryakbhyam Step by Step
📐 Vedic
Urdhva Tiryakbhyam for 2×2 digit multiplication — 3 passes: right column, crosswise, left column. Then for 3×3 digit — 5 passes. Each pass is a repeating pattern over digit positions. The loop count = number of digit positions!
💻 JS
for loops, while loops. Build a JS function that generates the full multiplication table up to n×n, displayed as an HTML table. Each table cell coloured by the digital root of the product — Vedic maths pattern visualised!
🎯 Outcome: A coloured multiplication table webpage where colour shows digital root patterns — visually demonstrating Vedic number harmony.
Week 10
JS Functions — Sutras as Reusable Code
📐 Vedic
A sutra is a rule you apply to any valid input. Ekadhikena works for 1/19, 1/29, 1/39... Nikhilam works for any numbers near any base. You don't rewrite the rule each time — you apply it. That's exactly what a function does.
💻 JS
function keyword, parameters, return. Write functions: digitalRoot(n), multiplyByEleven(n), squareEndingFive(n), nearBaseMult(a, b, base). Call each from the calculator. Students see that every Vedic sutra becomes one clean function.
🎯 Outcome: Students refactor their Week 8 calculator to use named functions — one per sutra. Code becomes readable, organised, and elegant — just like the sutras themselves.
Week 11
JS Arrays — Working with Digit Sequences
📐 Vedic
Many Vedic techniques work digit by digit: digital root adds each digit, Nikhilam operates per digit, the "casting out 9s" verification collects and sums digits. Digits of a number = a list = an array.
💻 JS
Arrays: [], .push(), .length, .forEach(), .map(), .reduce(). Convert number to array of digits: "1234".split("").map(Number). Build: sum of digits (digital root), Vedic digit manipulation, "Casting out 9s" verifier as a webpage.
🎯 Outcome: Students build a "Vedic Verifier" — enter any multiplication and the page confirms correctness using casting out 9s, showing each digit step.
Week 12
CSS Animations + JS — Making Vedic Patterns Move
📐 Vedic
Vedic art: yantras, mandalas, and rangoli all have mathematical patterns — rotational symmetry (multiples of 360°), reflective symmetry. Digit patterns in 9s table (9,18,27,36,45,54,63,72,81,90) form a visual symmetry when charted.
💻 CSS + JS
@keyframes, animation, transform: rotate(). JS: setInterval() to animate. Build a spinning Vedic yantra made of div elements. Each petal's position calculated from a Vedic proportional formula. Dynamic — spins and pulses.
🎯 Outcome: Students create an animated Vedic yantra in a browser using pure CSS and JS. A beautiful, share-worthy artefact that demonstrates both cultural and technical mastery.
Phase 4: Final Project — Teach What You Know Weeks 13–16
Week 13
Project Planning — Design Your Vedic Teaching Website
📐 Vedic
Students pick 2–3 Vedic sutras to teach on their website. Write out: Sanskrit name, English translation, worked example in 5 steps, 3 practice problems with answers. This becomes the content for their website.
💻 Web Design
Wireframe: Home page, Sutras page, Interactive Calculator page, About page. Sketch on paper first. Choose colour scheme. List which JS functions they'll need. Map content to HTML sections. One week, four pages planned.
🎯 Deliverable: Paper wireframes for all 4 pages, content written out, JS function list drawn up. Teacher signs off before building starts.

Project Build — Structure & Style All Pages
📐 Vedic

💻 HTML + CSS

🎯 By end: All 4 HTML pages exist, are styled, and navigation works. Content is accurate and readable.
Week 15
Project JS — Add All Interactive Elements
📐 Vedic
The interactive calculator must show Vedic steps, not just the final answer. Step 1, Step 2, Step 3 — each appears as the user enters input, mirroring how a Vedic scholar would teach a student. Teaching = debugging = verification.
💻 JS

Week 16 — Demo Day 🎉

Sample Exercises from the Program

These are the kinds of paired exercises students work through each week — a Vedic maths puzzle solved mentally, then the same logic coded immediately after.

Exercise A — Grades 4–5 (Scratch)

Week 4: ×11 Trick → Scratch Calculator

Vedic Part (Paper First — 5 min)
Solve these mentally using the ×11 sutra:
34 × 11 → Split 3_4, middle = 3+4 = 7 → 374
62 × 11 → Split 6_2, middle = 6+2 = 8 → 682
85 × 11 → Split 8_5, middle = 8+5 = 13, carry → 935
Now Code It in Scratch (30 min)
Build a ×11 calculator that:
• Asks for a 2-digit number
• Uses 3 variables: Tens, Units, Middle
• Sets Middle = Tens + Units
• If Middle < 10: says (Tens)(Middle)(Units)
• If Middle ≥ 10: carries the 1 to Tens
Extension Challenge
Modify the code to also check if the number IS divisible by 11 (digital root trick). Add a "🎉 Divisible!" message.
📎 Starter .sb3 file provided | Teacher answer key included

Exercise B — Grades 6–7 (HTML/JS)

Week 10: Functions → Sutra Library

Vedic Part (Paper First — 5 min)
Apply each sutra to the number pair (97, 96):
Nikhilam (base 100): deficits = 3, 4. Left = 93. Right = 12. → 9312
Digital root check: Root(9312) = 9+3+1+2 = 15 → 6. Root(97)×Root(96) = 7×6 = 42 → 6. ✓
Now Code It in JS (30 min)
function nikhilam(a, b, base) { let dA = base - a; let dB = base - b; let left = a - dB; let right = dA * dB; return left * base + right; } function digitalRoot(n) { return n === 0 ? 0 : 1 + (n-1) % 9; }
Extension Challenge
Write a verify(a, b) function that uses nikhilam() then checks with digitalRoot(). Return true/false and display ✓ or ✗ on the page using DOM manipulation.
📎 HTML starter template provided | Expected output screenshot included

Exercise C — Grades 4–5 (Scratch)

Week 9: Digital Roots → If/Else Divisibility

Vedic Divisibility Rules
Divisible by 9: digital root = 9
Divisible by 3: digital root = 3, 6, or 9
Divisible by 2: last digit even
Test: 738 → 7+3+8 = 18 → 1+8 = 9 → div by 9 AND 3!
Scratch Challenge
Build a "Vedic Divisibility Tester" that:
• Asks for any number
• Calculates its digital root using a variable
• Uses if/else blocks to check all three rules
• Announces all applicable divisors
• Works for any number the student enters
📎 Flowchart template provided | Visual step-by-step guide included

Exercise D — Grades 6–7 (CSS+JS)

Week 12: Vedic Pattern → Animated Yantra

Vedic Pattern — 9s Table Circle
Plot the units digit of each multiple of 9 on a clock-face: 9→9, 18→8, 27→7, 36→6... They go 9,8,7,6,5,4,3,2,1,0 — a perfect symmetrical pattern! Connect the dots → a star polygon. That's a yantra!
CSS + JS Build
/* 10 dots on a circle */ .dot { position: absolute; width: 12px; height: 12px; border-radius: 50%; background: gold; /* JS sets left/top using sin/cos of angle */ } /* Connecting lines drawn using SVG in JS */
Animation Layer
Add CSS @keyframes rotation so the entire yantra slowly spins. Make each dot pulse. The result is a live, animated Vedic mathematical artwork — generated entirely by student code.
📎 Starter HTML with SVG setup provided | Video walkthrough for teacher
